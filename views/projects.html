HydroSource

eCommerce Website | MVC Single Page Application | MEAN stack
HTML5 CSS Compass/SASS BootStrap UI Bootstrap JavaScript Angular.js Node.js Express.js MongoDB/Mongoose

HydroSource is an eCommerce website for a (fake) water subscription service. A unique feature is Water Intake Calculator, which takes a user's body weight and hydration level of choice, and calculates the ideal amount of daily water intake. Based on the daily intake and delivery frequency of choice, total water supply for each delivery is determined. It is displayed in Store and Cart on a progress bar, to aid a uer's shopping experience.

What I learned

bcrypt (npm)To avoid potential security risks, it is a common practice to have a user's password hashed before saving it in a database. I learned the basic concept of bcrypt, nad how to read a shadow password (prefix for bcrpy hash, cost parameter, salt and hash). I used a Node.js library to initially hash a password and to verify a password given at sign in.

rand-token (npm)Another Node.JS library that I utilized in developing HydroSource. A random token is generated when a user signs in, and is removed at sign out.

Authentification CookiesAs a user, sign in/out status used to feel as if it was a distinct state, like on and off. While implementing it, I realized it was rather a set of actions that is initiated by one time verification of username and password, followed by creation of a token, which is used for repeated verifications on every page access (or route change, to be precise). Use of cookies is essential in the process, for it allows a token to be stored and be accessible throughout the website.
Although it was technically simple implementation, it was intriguing to experience a change in the concept of sign in/out - a perspective shift from a user to a developer's side.

Stripe (API)Stripe is an online payment software platform. By using the Stripe API that has NodeJS specific library, I can process payments safely without dealing with a user's sensitive information.

Problems Encountered and Solved

Use of ng-show/ng-hide in navigation menu itemsIt was an architectural problem in using Angular routing. I used ng-show/ng-hide to display or hide menu items (in index.html) depending on sign in/out status. The problem was that they were evaluated only once at an initial page load. Whether a user signs in or not after the initial loading, because it is routing between different views not reloading the page, ng-show/ng-hide on menu items were not being updated. To solve the problem, I added a run block .run() with $rootScope and $location : a global variable $rootScope.loggedIn is used to determine sign in/out status, shared by all controllers, and $locationChangeStart monitors route change.

Marti's Study Room

Korean Vocaburary Study Web App | MVC Single Page Application | MEAN stack
HTML5 CSS JavaScript Angular.js Node.js Express.js MongoDB/Mongoose

Marti's Study Room is an web application created for my friend, Marti, whom I teach Korean, to aid her vocaburary memorization. I used to send her emails with a quiz file attached and wanted to automate this process to save time. With the app, Marti can take a quiz whenver she wants, and I can track her progress.

Find Your Festival

One-stop Search Web App for music festivals | MVC Single Page Application
HTML5 CSS JavaScript Angular.js Node.js Express.js MongoDB/Mongoose

Find Your Festival is an one-stop solution for festival-goers who want to decide which music festival(s) to attend. A user provides search criteria and Find Your Festival returns matches with extimated cost, or even a complete itinerary. It also comes with options to buy a ticket and book a flight or a hotel.
