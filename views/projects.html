Mini City

1st Place at Goodie Nation Hackathon | NFC Tag Based Data Management Portal | Python and MySQL
HTML5 CSS Compass/SASS BootStrap Jinja 2 Python Flask MySQL/Sequel Pro

My team won first place at the Goodie Hackathon in the General category. We built the basic set-up for a data management portal for our client, Mini City, a non-profit organization based in Atlanta helping the homeless. Mini City wanted to provide services using Near Field Communication (NFC) tag based wearable devices and to manage data gathered from them.
Work in progress: Currently updating the MVP from the Hackathon for my final project at DigitalCrafts

NFC Gained familiarity in NFC technology and its application in mobile devices. Created a NFC tag writer which generates and assigns a serial number to a new NFC tag when scanned.

Passenger and Python WSGI I deployed the web app using Phusion Passenger to a Dreamhost server the Hackathon had set up for each team.

HydroSource

eCommerce Website | MVC Single Page Application | MEAN stack
HTML5 CSS Compass/SASS BootStrap UI Bootstrap JavaScript Angular.js Node.js Express.js MongoDB/Mongoose
Test with username test, password 1111111111

HydroSource is an eCommerce website for a (fake) water subscription service. A unique feature is Water Intake Calculator, which takes a user's body weight and hydration level of choice, and calculates the ideal amount of daily water intake. Based on the daily intake and delivery frequency of choice, total water supply for each delivery is determined. It is displayed in Store and Cart on a progress bar, to aid a user's shopping experience.

What I Learned

bcrypt (npm) I used a Node.js library to hash a password before saving it in the database.I learned the basic concept of bcrypt, how to read a shadow password (prefix for bcrypt hash, cost parameter, salt and hash).

rand-token (npm) Another Node.js library I used. A random token is generated when a user signs in, and is removed at sign out.

Authentication CookiesAs a user, sign in/out status used to feel as if it was a distinct state, like on and off. While implementing it, I realized it was rather a set of actions that is initiated by one time verification of username and password, followed by creation of a token, which is used for repeated verifications on every page access (or route change, to be precise). Use of cookies is essential in the process, for it allows a token to be stored and be accessible throughout the website. Although it was technically simple implementation, it was intriguing to experience a change in the concept of sign in/out - a perspective shift from a user's to a developer's side.

Stripe (API)Stripe is an online payment software platform. I used Stripe's Node.js library to process payment.

Problems Encountered and Solved

Use of ng-show/ng-hide in navigation menu itemsIt was an architectural problem in using Angular router. I used ng-show/ng-hide to display or hide menu items depending on sign in/out status. The problem was that they were evaluated only once at the initial page load. To solve the problem, I added a run block .run() with $rootScope and $location . A global variable $rootScope.loggedIn was used to indicate sign in/out status and $locationChangeStart to monitor route change.

Marti's Study Room

Korean Vocaburary Study Web App | MVC Single Page Application
HTML5 CSS JavaScript Angular.js Node.js

Marti's Study Room is a web application that I created for a student I tutor. She wants to get tested on her vocabulary on a regular basis. Instead of writing a quiz each session, I wanted to automate the process so that she can take the quiz whenever she wants.
Work in progress: planning to implement a database connection

What I Learned

Foreign Language Characters When I was working locally, Korean characters were broken, looking like a gibberish. When I served it up on Node.js, they started to display correctly. Researching why, I learned that there were different types of encoding, depending on the language's character sets. I understand why UTF-8, not ASCII, is now widely used. Node.js natively supports UTF-8 encoding, which was why Korean characters were displayed correctly once served up by Node.js. I also added <meta charset="UTF-8"> to make sure web browsers correctly encode my scripts.

Problems Encountered and Solved

Logic for generating quiz problems and answer options t was more complicated than I initially thought to write a program that randomly selects a word and pairs it with the answer options, only one of which is the correct answer, and track each one of them. It was when I did not know how to use a database; I simply had a .json file that contained vocabulary as an object. I ended up using multiple arrays to temporarily store different groups of words (e.g., problems, correct answer options, incorrect answer options). I am planning to connect a database with a schema, which will help simplify the process of generating quiz questions.
