Mini City

1st Place at Goodie Nation Hackathon | NFC Tag Based Data Management Portal | Python and MySQL
HTML5 CSS Compass/SASS BootStrap Jinja 2 Python Flask MySQL/Sequel Pro

I participated in Goodie Nation Hackathon, October 14-16, with two other developers. Mini City is a group my team worked with, whose goal is to address the issue of homelessness in Atlanta. They wanted to utilize NFC-based wearable wristbands to create and manage data. My team built the basic set up for their data management portal, and won the first place in General category.

What I learned

NFC I learned what Near Field Communication is and how Android makes use of the technology. Using Android API, my team made a NFC writer as a mobile app, which scans a new NFC tag and assigns a randomly generated serial number to it.

Passenger and Python WSGI I deployed the web app using Phusion Passenger to a Dreamhost server the organizer had set up for each team.

Problems Encountered and Solved

Working with real life clients Working with Mini City was vastly different from working on an in-class or a personal project. They had only the general idea without technical consideration and were not familiar with any of technology. Translating the vague idea into the technical concepts and developmental steps was challenging at first, but once it became clear that it needed to start from the scratch, the rest was enjoyable.

HydroSource

eCommerce Website | MVC Single Page Application | MEAN stack
HTML5 CSS Compass/SASS BootStrap UI Bootstrap JavaScript Angular.js Node.js Express.js MongoDB/Mongoose

HydroSource is an eCommerce website for a (fake) water subscription service. A unique feature is Water Intake Calculator, which takes a user's body weight and hydration level of choice, and calculates the ideal amount of daily water intake. Based on the daily intake and delivery frequency of choice, total water supply for each delivery is determined. It is displayed in Store and Cart on a progress bar, to aid a uer's shopping experience.

What I learned

bcrypt (npm) I used a Node.js library to hash a password before saving it in the database.I learned the basic concept of bcrypt, how to read a shadow password (prefix for bcrpy hash, cost parameter, salt and hash).

rand-token (npm) Another Node.js library I used. A random token is generated when a user signs in, and is removed at sign out.

Authentification CookiesAs a user, sign in/out status used to feel as if it was a distinct state, like on and off. While implementing it, I realized it was rather a set of actions that is initiated by one time verification of username and password, followed by creation of a token, which is used for repeated verifications on every page access (or route change, to be precise). Use of cookies is essential in the process, for it allows a token to be stored and be accessible throughout the website.
Although it was technically simple implementation, it was intriguing to experience a change in the concept of sign in/out - a perspective shift from a user to a developer's side.

Stripe (API)Stripe is an online payment software platform. I used Stripe's Node.js library to process payment.

Problems Encountered and Solved

Use of ng-show/ng-hide in navigation menu itemsIt was an architectural problem in using Angular router. I used ng-show/ng-hide to display or hide menu items (in index.html) depending on sign in/out status. The problem was that they were evaluated only once at an initial page load and will not chance after a user sings in later. To solve the problem, I added a run block .run() with $rootScope and $location . A global variable $rootScope.loggedIn was used to determine sign in/out status, shared by all controllers, and $locationChangeStart to monitor route change.

Marti's Study Room

Korean Vocaburary Study Web App | MVC Single Page Application | MEAN stack
HTML5 CSS JavaScript Angular.js Node.js Express.js MongoDB/Mongoose

Marti's Study Room is an web application created for my friend, Marti, whom I teach Korean, to aid her vocaburary memorization. I used to send her emails with a quiz file attached and wanted to automate this process to save time. With the app, Marti can take a quiz whenver she wants, and I can track her progress.

What I learned

bcrypt (npm)To avoid potential security risks, it is a common practice to have a user's password hashed before saving it in a database. I learned the basic concept of bcrypt, nad how to read a shadow password (prefix for bcrpy hash, cost parameter, salt and hash). I used a Node.js library to initially hash a password and to verify a password given at sign in.
